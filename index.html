Sherpa

Sherpa stands for Simple HTTP RPC API. It allows you to easily (trivially!) export server-side functions over HTTP, for use in your web application, much like JSON-RPC. Some features of Sherpa API's:

  • Simple to create on server-side.
  • Simple to use on client-side, especially using JavaScript.
  • Self-describing: an API lists available functions and provides documentation.

Sherpa API's provide a little bit of JavaScript glue that make it extremely easy to start using them. An example: 

<script src="https://sherpa.irias.nl/exampleapi/sherpa.js">
<script>
	window.exampleapi.requestCount()
	.then(function(result) {
		alert('Number of requests: '+result);
	}, function(error) {
		alert('Error: '+JSON.stringify(error));
	});
</script>

Sherpaweb

Sherpaweb is a simple web app that renders the documentation provided by Sherpa API's. Given a Sherpa API URL, sherpaweb will:

  • Show you the API documentation.
  • Let you call API functions with parameters you provide.

Discovering the functionality of an API was never this easy.

A public sherpaweb instance is running at sherpa.irias.nl. It will work with any Sherpa API that your browser has access to.

Open sherpa.irias.nl/x/sherpa.irias.nl/exampleapi/ to look at sherpaweb's builtin example API.

Implementing a Sherpa API

Is easy:

  1. Implement the functions you want to export in your programming language of choice.
  2. Pick (or create) a Sherpa API server implementation.

For a real-world usage example, see sherpaweb's exampleapi.go. The API documentation is written as Go comments, just like you would normally write comments for your code. The sherpa library for Go comes with a tool called sherpadocs that will turn those comments into a JSON file suitable for use as API documentation. This makes it effortless to write and publish documentation for your API's.
