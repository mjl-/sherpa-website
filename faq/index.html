FAQ

Are Sherpa API's RESTful?

Not really. REST API's take a data-centric approach. They try to model the data on URL's, and use HTTP methods to model operations. Sherpa API's take a function centric approach, just like JSON-RPC and (a longer time ago) XMLRPC. Sherpa API's takes away all the trouble of coming up with proper URL mappings, HTTP method mappings and header intricacies (typically simply ignored by REST API designers).

API versioning

Clients should check that they support the Sherpa protocol version specified in metaVersion. If a future Sherpa protocol is incompatible, it will probably be easiest to just use a new (additional) API endpoint to export your API functions using the new protocol.

You'll probably mostly increase the minor and patch version of the semantic version of your API. In practice, it is very hard to remove API functions or make backwards incompatible changes. This is usually done by either publishing a new API on a new endpoint, or by adding new methods (and possibly marking old ones as deprecated).
